# Bebimom 모노레포 가이드

## 적용 범위
- 이 문서는 `frontend/web`, `frontend/mobile`, `frontend/shared`에 공통으로 적용되는 규칙을 정의합니다.
- 앱별 구현 세부 규칙은 각 디렉터리의 `CLAUDE.md`에서 관리합니다.

## 아키텍처 경계
- `frontend/shared`는 공용 타입, 상수, 토큰의 단일 출처로 사용합니다.
- 앱(`web`, `mobile`)은 `shared`를 import할 수 있지만, `shared`에서 앱 폴더를 import하면 안 됩니다.
- 유사한 타입/인터페이스를 중복 정의하기 전에 공용 계약으로 올릴 수 있는지 먼저 검토합니다.

## Import와 경로
- 설정된 경우 alias import(`@/...`, `@shared/...`)를 우선 사용합니다.
- alias가 있는 경로에서 깊은 상대경로 사용은 지양합니다.
- 경로 대소문자와 폴더 네이밍(`constant` vs `constants`)은 하나로 통일해 유지합니다.

## 환경 변수
- API base URL이나 민감값을 코드에 하드코딩하지 않습니다.
- 클라이언트 노출 변수는 플랫폼 규칙을 따릅니다.
  - Expo: `EXPO_PUBLIC_*`
  - Next.js: `NEXT_PUBLIC_*`
- 서버 비밀값은 클라이언트 노출 env에 절대 넣지 않습니다.

## 라우팅/네비게이션
- 화면 곳곳에 라우트 문자열을 하드코딩하지 않습니다.
- 반복 경로는 라우트 상수로 관리합니다.
- 라우트 이름은 의미 기반으로 정하고 한곳에서 관리합니다.

## 상태와 인증
- 민감 토큰은 플랫폼 보안 저장소에 저장합니다(모바일: SecureStore).
- 메모리 인증 상태는 최소 필드(`user`, `accessToken`, `isLoading`)로 명확히 유지합니다.
- hydrate 로직은 실패해도 안전하게 종료되며 로딩 상태를 반드시 해제해야 합니다.

## 코드 품질
- API 레이어는 명시적 응답 타입과 strict TypeScript를 우선합니다.
- UI 상수/토큰은 인라인 반복 대신 중앙 관리합니다.

## 작업 방식
- 변경은 작고 목적 중심으로 나눠서 진행합니다.
- 수정 파일은 가능하면 lint/type check로 검증합니다.
- 비밀값이나 로컬 `.env` 파일은 커밋에 포함하지 않습니다.
