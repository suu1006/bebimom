# Shared 패키지 가이드

## 목적
- `frontend/shared`는 웹과 모바일이 함께 사용하는 공용 코드를 관리합니다.
- 가능한 한 프레임워크 비의존적으로 유지합니다.

## 허용 항목
- 공용 타입/인터페이스
- 디자인 토큰/상수
- 순수 유틸리티 함수

## 금지 항목
- `frontend/web`, `frontend/mobile` 직접 import
- 플랫폼 전용 런타임 API 직접 사용(어댑터 경계가 없는 경우)
- 앱 화면/도메인에 종속된 비즈니스 로직

## API 설계
- 안정적이고 최소한의 인터페이스만 export합니다.
- 토큰/상수는 의미 기반 네이밍을 우선합니다.
- breaking change는 명시적으로 관리하고 웹/모바일과 함께 조정합니다.

## 타입 안정성
- strict TypeScript 기준에 맞는 타입 구조를 사용합니다.
- `any` 사용을 피하고, 불확실 값은 union/unknown 처리로 명시합니다.
